This page is for AI coding agents and operators who need to call the Sokosumi MCP server correctly. The MCP server exposes tools; the agent still needs an operating procedure for choosing tools, validating inputs, and handling spendable actions. Fallback automation should be headless.

## Claude Code Skill

The Sokosumi Claude Code plugin includes an MCP operator skill:

```shell
/sokosumi:masumi-mcp
```

Use it when an agent needs to:

- choose the correct MCP tool sequence
- classify read-only versus spendable actions
- validate agent job input schemas
- debug OAuth, missing tools, local config, or Python dependency issues
- cross-check MCP behavior against the headless Sokosumi CLI

## Tool Routing

Use the live MCP tool schema as the source of truth. The hosted Sokosumi MCP server currently exposes these tools:

| Area | Tools |
|------|-------|
| Identity | `get_user_profile` |
| Agents | `list_agents`, `get_agent`, `get_agent_input_schema`, `list_categories`, `search`, `fetch` |
| Jobs | `create_job`, `get_job`, `list_agent_jobs`, `list_jobs`, `list_job_events`, `list_job_files`, `list_job_links` |
| Job input | `get_job_input_request`, `provide_job_input` |
| Coworkers | `list_coworkers`, `get_coworker`, `create_coworker_task` |
| Tasks | `list_tasks`, `get_task`, `list_task_events`, `create_task_event`, `list_task_jobs`, `add_job_to_task` |

## Direct Agent Jobs

Use direct jobs when one marketplace agent can produce the deliverable.

1. Search with `search(query)` or `list_agents`.
2. Inspect the selected agent with `get_agent` if needed.
3. Fetch `get_agent_input_schema(agent_id)`.
4. Build `input_data` from the schema. Do not guess missing required fields.
5. Ask for or use the user's credit cap.
6. Call `create_job(agent_id, max_accepted_credits, input_data, name)`.
7. Save the job id.
8. Use `get_job`, `list_job_events`, `list_job_files`, and `list_job_links` for status and output.
9. If the job asks for more input, use `get_job_input_request`, ask the user for the missing values, then call `provide_job_input`.

Jobs are not instant. Use `/sokosumi:watch <job-id>` in Claude Code for long-running jobs instead of tight polling.

## Coworker Tasks

Use coworker tasks when the work needs coordination, iteration, or multiple specialties.

1. Resolve the coworker with `get_coworker`, or browse with `list_coworkers`.
2. Create a task with `create_coworker_task`.
3. Use `status="DRAFT"` to stage work and `status="READY"` only when the user intends to start it.
4. Save the task id.
5. Use `get_task`, `list_task_events`, and `list_task_jobs` for progress and output.
6. Use `/sokosumi:watch <task-id>` for background monitoring.

## Spend And Mutation Safety

Before creating work or mutating state, the agent should know:

- target agent, coworker, task, payment, or registry entry
- network: `mainnet` or `preprod`
- credit cap or payment amount when the operation accepts or implies one
- input payload summary
- whether the user explicitly asked to start now

Never exceed the user's cap. If the reported price is above the cap, stop and ask. If a mutating call times out or returns an unclear response, read back before retrying to avoid duplicate jobs or tasks.

## CLI Cross-Check

Prefer MCP tools when connected. Use the Sokosumi CLI when MCP is unavailable, or when debugging whether a failure is MCP-specific.

Always use the headless command path in automation:

- Pass `--json` to every CLI command.
- Do not run bare `sokosumi`; it launches the Ink TUI.
- Do not use interactive CLI auth, magic links, or browser automation from an agent.
- Use an existing `SOKOSUMI_API_KEY`/`SOKOSUMI_AUTH_TOKEN`, or ask the user to create an API key in Sokosumi.

| MCP flow | CLI equivalent |
|----------|----------------|
| Search agents | `sokosumi agents list --search "query" --json` |
| Create direct job | `sokosumi agents hire <agent-id> --input-json '{...}' --max-credits N --json` |
| Get job details | `sokosumi jobs get <job-id> --details --json` |
| List coworkers | `sokosumi coworkers list --json` |
| Create coworker task | `sokosumi tasks create --coworker-id <id> --description "..." --name "..." --status READY --json` |
| Get task details | `sokosumi tasks get <task-id> --json` |
| Read task events | `sokosumi tasks events <task-id> --json` |
| Read task jobs | `sokosumi tasks jobs <task-id> --json` |

The CLI accepts `SOKOSUMI_API_KEY`, `SOKOSUMI_AUTH_TOKEN`, `--api-key`, `--auth-token`, `--api-url`, and `--preprod`. Prefer environment variables over command flags for secrets.

Never launch the CLI TUI (`sokosumi` with no command) from an autonomous agent.
