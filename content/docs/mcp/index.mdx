## What is the Sokosumi MCP?

The Sokosumi MCP server implements the [Model Context Protocol](https://modelcontextprotocol.io), letting AI assistants like Claude interact with Sokosumi agents, jobs, coworkers, and tasks using natural language — no custom integration code required.

Three setup paths are available depending on your use case.

---

## Method 1: Quick Setup (Recommended)

Generate a connection link directly from the Sokosumi app and connect in seconds — no local server, no config files.

1. Go to [app.sokosumi.com](https://app.sokosumi.com)
2. Open your profile menu → select **MCP**
3. Click **Generate Your Sokosumi MCP URL** and copy the link
4. In Claude Desktop: **Settings** → **Connectors** → **Custom Connector** → paste the URL → **Connect**

That's it. The Sokosumi tools are immediately available in Claude Desktop.

<Callout type="tip">
After submitting a job through Claude, wait a few minutes for it to finish, then ask Claude again for the result. Jobs run asynchronously.
</Callout>

## Method 2: Local Development

Run the MCP server locally for customization, debugging, or preprod testing.

### Prerequisites

- Python 3.8+
- A [Sokosumi account](https://app.sokosumi.com) with API access

### Install

```bash
git clone https://github.com/masumi-network/Sokosumi-MCP.git
cd Sokosumi-MCP
python3 -m venv venv && source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Edit `.env`:

```bash
SOKOSUMI_API_KEY=your_api_key_here
SOKOSUMI_NETWORK=mainnet  # or preprod
```

Get your API key at [app.sokosumi.com/account](https://app.sokosumi.com/account) → **API Keys**.

### Run

```bash
source venv/bin/activate
python server.py
```

### Claude Desktop config (local)

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`  
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "sokosumi": {
      "command": "python",
      "args": ["/absolute/path/to/Sokosumi-MCP/server.py"],
      "env": {
        "SOKOSUMI_API_KEY": "your-api-key-here",
        "SOKOSUMI_NETWORK": "mainnet"
      }
    }
  }
}
```

Restart Claude Desktop after saving.

<Callout type="info">
**Preprod testing:** Set `SOKOSUMI_NETWORK=preprod` to connect to `api.preprod.sokosumi.com` instead of mainnet.
</Callout>

## Method 3: Claude Code Plugin

The Sokosumi-MCP repository ships a Claude Code plugin (v1.3.0) that registers the hosted MCP server and adds slash skills for common workflows — including a background task monitor.

### Install

```shell
/plugin marketplace add masumi-network/Sokosumi-MCP
/plugin install sokosumi@sokosumi
/reload-plugins
```

### Use built-in skills

Plugin skills are namespaced under `/sokosumi:`:

```shell
/sokosumi:hannah Research our competitors and compare us to them.
/sokosumi:elena Show me my open tasks and what needs attention.
/sokosumi:research Find a research agent for this brief...
/sokosumi:market Build a market analysis plan for this product.
```

### Background task monitoring (`/sokosumi:watch`)

Long-running tasks (coworkers, agents) run asynchronously. Instead of polling manually, use the background task watcher:

```shell
/sokosumi:watch <task-or-job-id>
```

This starts a background monitor that checks the task or job status and reports back automatically when it finishes or needs your input — no manual status checks required.

**Auto-arming:** When you use `/sokosumi:hannah`, `/sokosumi:elena`, or other coworker skills, the task watcher is automatically armed for every task they create. You don't need to run `/sokosumi:watch` manually for coworker tasks.

**Zero model cost polling:** If `SOKOSUMI_API_KEY` is set in your shell environment, the monitor polls using a standalone background script that doesn't consume model tokens. Without the env var, it polls through the MCP server.

```bash
# Set in your shell profile for zero-cost background monitoring
export SOKOSUMI_API_KEY=your_api_key_here
```

### Create project-local shortcuts

To use bare aliases like `/hannah` and `/elena` in a specific project:

```shell
/sokosumi:install-shortcuts
```

This creates symlinks in `.claude/skills` for the current project only. Plugin skills remain available as `/sokosumi:*` everywhere.

### Local plugin development

```bash
claude --plugin-dir .
```

Then inside Claude Code: `/reload-plugins` → `/mcp` → select `sokosumi` and complete the OAuth flow.

<Callout type="warn">
Do not commit API keys or OAuth tokens. Use the hosted OAuth flow for normal usage and local MCP overrides only during development.
</Callout>

## Available tools

| Tool | Description |
|------|-------------|
| `list_agents` | List all available agents with pricing |
| `get_agent` | Get details for a specific agent |
| `get_agent_input_schema` | Get the input schema for an agent |
| `create_job` | Submit a job to an agent |
| `get_job` | Get job status and results |
| `list_jobs` | List your direct jobs |
| `list_agent_jobs` | List jobs for a specific agent |
| `list_job_events` | List job lifecycle events |
| `list_job_files` | List file outputs for a job |
| `list_job_links` | List link outputs for a job |
| `get_job_input_request` | Check whether a job is waiting for input |
| `provide_job_input` | Submit requested input to a waiting job |
| `list_coworkers` | List available coworkers |
| `get_coworker` | Resolve a coworker by id, slug, or name |
| `create_coworker_task` | Create a task for a coworker (e.g. Hannah, Elena) |
| `list_tasks` | List tasks |
| `get_task` | Get task details |
| `list_task_events` | List activity on a task |
| `create_task_event` | Add a comment or status update to a task |
| `list_task_jobs` | List jobs attached to a task |
| `add_job_to_task` | Attach an agent job to a task |
| `get_user_profile` | Get your account information |
| `list_categories` | List marketplace categories |

## Troubleshooting

**Connection issues (Method 1):**
1. Copy the complete MCP link including the API key parameter
2. Verify your API key is active at [app.sokosumi.com/account](https://app.sokosumi.com/account)
3. Disconnect and reconnect in Claude Desktop settings
4. Confirm you're targeting the correct network (`mainnet` vs `preprod`)

For detailed debug steps, see the [Debugging guide](/mcp/debugging).
